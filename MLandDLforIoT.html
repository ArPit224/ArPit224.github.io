Abstract: Machine Learning and Deep learning are a part of Artificial Intelligence, An Umbrella term used for programs mimicking human intelligence for decision making and problem solving using Computers and other machines.

Machine Learning grants a computer ability to learn what it’s not coded to do exactly, i.e. from experience.  When multiple layers of neural networks are used to train a machine learning model, It’s called deep learning.

IoT is collection of objects which are connected together wirelessly or with data busses, where various inputs are assessed and a decision is made in accordance to pre-programmed behaviour or based on some pre-trained decision model.

IoT includes environmental and surrounding awareness through the use of various sensors which collects data in form which is very easy to process in an Artificial intelligence setup,

Either for training on that dataset or for decision making based on a pre-trained model.

AI powered systems are used in Medical Prognosis, Aircraft AutoPilots, Network Security, etc.

Introduction:

In all IoT systems there is a layer of decision making which can be enhanced to be more interactive, efficient, and adaptable using Machine Learning and Deep learning, as they can be trained to ignore erroneous data points, or making a more meaningful outcome out of given input, etc.

One of the major advantages of AI over Hard coded logical equivalent is that once trained, an AI can work with variation of input data while Hard Coded Logic based system only works with exact input it was coded for.

Training ML and DL for IoT

  1. Data Acquisition

Measuring real world data and converting it into digital values through the processes of sampling and quantization for further processing is called data acquisition. The components of a DAC systems are:

  • Sensors, A transducer which converts a physical stimulus into electrical signals
  • Signal conditioning circuits, Generally LPF, used for removing highly erroneous and garbage data for the purposes of further conversion

Figure 1: DAQ System

  • ADC, Analog-to-digital converters converts analog signal into digital values

Data Acquisition is done through either general purpose softwares like  LabView, Lisp, Java, C++, etc. or through open stand alone DAQ softwares called Data Loggers.

DAQ sensors can record various environmental variables such as, but not limited to, Color, Pressure, orientation, Movement, etc.

Once the signal is converted into digital representation it can be processed and worked upon by the machine for further feature extraction and learning.

  1. Data Processing

The collected data is often not in the form that is usable in machine algorithms, Therefore it needs to be processed before it's passed on to the model. Processing data for ML/Dl includes feature extraction, Null points removal, Conversion into the form that is understandable to an algorithm, For example conversion of a photo into a n x n Matrix.

  1. Machine Learning

In machine learning, a computer is trained to imitate human ability of decision making and intelligence. Machine learning is of two types:

  1. Supervised Learning

In supervised learning the correct label is present in the dataset, in training the model, the algorithm estimates prediction from input, adjusts for error and reiterates the process until desired accuracy is reached. Difference between true value and the predicted value, adjusted for desired bias, is called Cost function. There are many techniques by which we minimise the cost function of supervised learning, for example, stochastic gradient descent, momentum based gradient descent, etc.

Algorithms of Supervised learning are:

        1: Linear regression

        2: Logistic Regression

        3: Support Vector Machines

        4: Naïve Bayes

        5: KNN

        6: Random Forest

Supervised learning is used when the clear outcome of a dataset is known, i.e, it just to be classified or simply predicted.

Example: Supervised learning Can be used in a classification model that is when a data set is to be classified into Two or more categories, For example whether a message is spam or not.

  1. Unsupervised learning

In unsupervised learning, there is no outcome label predefined, Instead the algorithm automatically clusters the result into categories on its own.

Unsupervised learning algorithms are classified into two types>

1: Clustering

2: Association

Clustering means The data is ClusteredInto related clusters while association means Finding the rules between 2 or more events in the data, For example likelihood of Happening of X if Y has already happened. It is used in Suggestion of content  or shopping suggestions based on past activities.
Unsupervised Algorithms for unsupervised learnings are:

        1: K-means clustering

        2: Ward linkage

        3: Complete linkage

        4: Single Linkage

        5: Guassian mixture model

Unsupervised learning is used when pattern recognition is the main goal of a given scenario.

  1. Deep Learning

In deep learning the machine can process raw data itself, by automatically finding the features in it. Each representation, auto generated through the machine itself, represents an individual level. This allows DL models to be trained with data in its natural form leading to better integration between humans and the machine.

Applications of ML/DL in IoT

1: Health care assistance

ML/DL powered IoT is specifically useful in the health sector as there is both a dire need of automated systems which free up healthcare workers as well as abundance of data crunching jobs which can be easily automated using AI, Such as classification of cancer cells, monitoring of Old Age people, Vitals monitoring, etc.

  1. ML based health monitoring

Health problems affect everyone but at times it’s not possible to judge the severity of the symptom without consultation of a professional doctor, Which ties up the crucial resources in healthcare as the important infrastructure that can be used for helping ill patients is wasted on relieving the concerns of a healthy individual.

More importantly, it drives up the cost of medical help in private healthcare and locks out the poorer people out of the system.

This problem can be easily alleviated by the use of IoT - DL based systems which can provide decent prognosis based on sensors present on a mobile device alone.

This not only frees up the extreme working hours of healthcare workers but also provides us with a portable system which can be easily and quickly deployed on ground zero regardless of presence of infrastructure, and for most intents and purposes is infinitely reproducible.

When modern technology is used to collect, process, and deliver health care services, example use case could be: teleconsultation, telediagnosis, and remote surgery training.

Telemedicine is widely implemented in China to plug the gap between access to healthcare between urban and rural areas.[a]

Case 1: In endemics

 Use of this technology is particularly attractive in dealing with epidemics as it reduces the physical contact as well as it can be deployed quickly and in large quantities in a proactive rather than reactive manner.

Case 2: Identification of high risk individuals based on daily activities and habits

As modern day humans further integrate with technology, it’s very easy to document every physical/non-physical activity a person does daily using on-body sensors in smartphones, smart watches, and health monitoring gadgets.

As there is a strong correlation between lifestyle and many health problems such as … whatever The constant data collection can be leveraged for prediction, identification, and possible issuing of warning to a person whose current lifestyle puts them at serious health risk.

Obviously there is a huge privacy concern but if regulated and maintained by the government,  potential reduction in overall health cost, increase in living standard, and increase in productivity makes it an extremely lucrative technology to invest in.

2: Smart Home

In smart homes there are varying degrees of interconnected devices which work and share data together for enhanced efficiency and comfort. This can help in security, management, ease of living, energy saving, etc. Google home is one such example.

Privacy issues, and Single point for attack makes smart homes a troublesome Idea for some.

Case 1: Smart Connected devices

There are many devices in Modern homes that have some kind of electronic decision-making capabilities so by connecting them to each other a kind of Symphony Can be orchestrated for example dimming of light according to the sunlight, automatically cooking breakfast and Turning on television for News In the morning.

While these are Fairly simple Tasks to be automated using basic electronic logic devices, By Using machine learning The house can be personalised according to the need of the individual As in automatically ordering the desired food Based on weather condition all specific mood.

Case 2: Smart Grid

In a power grid  peak production of electricity often does not match up with peak consumption, And storing electricity For a long time is often  not an economically effective idea. Therefore if the production company can predict beforehand when the consumption will peak they can work the production capacity  around  that time.

Another Way to leverage deep learning in this scenario would be remotely turning on and off air conditioning of the house so that  its active time can be offloaded to that time of day when Demand for electricity is lower So that There is  decreased probability of grid failure Without causing much discomfort to the user.
